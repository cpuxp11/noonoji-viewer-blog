"눈오지야, 안녕."

단톡방에 한 줄 던졌다.
1분이 지났다. 답이 없다.
2분, 3분, 4분.

"느린데?"
나는 5분을 기다렸다.

---

## 봇이 왜 이렇게 느려졌나

게이트웨이 로그를 깠다.

```
stuck session: sessionId=noonoji age=305s queueDepth=1
workspace bootstrap file AGENTS.md is 12951 chars (limit 12000); truncating
```

`age=305s`. 5분 동안 한 메시지를 처리하고 있다는 뜻이다.
그 위에 한 줄이 더 있었다. "AGENTS.md가 12,951자라 한도(12,000)를 넘어서 잘라냈음."

세어 봤다.

| 파일 | 사이즈 |
|------|--------|
| **눈오지 AGENTS.md** | **17,303자** |
| 시고르 AGENTS.md | 8,476자 |
| 저당근 AGENTS.md | 3,009자 |

저당근은 3K, 시고르는 8K. 눈오지만 17K.

---

## 17,000자에 뭐가 들었나

`AGENTS.md`는 봇이 매 세션 시작할 때 자동으로 머리에 부어넣는 파일이다.
즉, **단톡에서 "안녕" 하나 답하려고 17K짜리 매뉴얼을 매번 다시 읽는다.**

뭐가 들었는지 봤다.

| 섹션 | 줄 수 |
|------|-------|
| 💓 Heartbeats - Be Proactive! | 81 |
| Memory | 49 |
| 성능 향상 규칙 (답변 품질) | 47 |
| Group Chats | 46 |
| 역할 분리 (시고르/저당근/다미) | 35 |
| RTK (토큰 절약) | 15 |
| Tools | 13 |
| ... | ... |

총 15개 섹션. 전부 "매번" 들고 다닌다.

단톡 "안녕"에 답하는데 _옵시디언 볼트 다루는 규칙_ 까지 같이 들어간다는 얘기다.

---

## 사람도 똑같다

회의 한 번 들어가는데, 회사 매뉴얼 전체를 머리에 넣고 들어가는 사람은 없다.
"오늘 회의는 일정 조율이니까 캘린더 룰만 챙기면 됨."
필요한 것만 들고 들어간다.

근데 우리는 봇한테 그렇게 안 시켰다.
"매뉴얼 다 외워서 들어가."
당연히 굼뜰 수밖에.

---

## Thin Harness — 가벼운 골격

해결책은 단순했다.
17K짜리 한 파일을, **항상 들고 다니는 4.7K + 컨텍스트별 사이드카**로 쪼갰다.

| 파일 | 사이즈 | 언제 로드 |
|------|--------|-----------|
| AGENTS.md | 4,693자 | 항상 (자동) |
| GROUP.md | 2,317자 | 그룹챗에 들어갈 때만 |
| HEARTBEAT.md | 2,595자 | cron 깨우기 때만 |
| QUALITY.md | 2,614자 | 복잡한 답변 만들 때만 |
| ROLES.md | 2,091자 | 봇끼리 라우팅할 때만 |
| MEMORY-RULES.md | 2,919자 | 메모리 쓸 때만 |

AGENTS.md는 "라우터"가 되었다.
"그룹챗에 들어갔니? 그럼 GROUP.md 읽어. heartbeat이니? HEARTBEAT.md 읽어."

이러면 단톡 "안녕"에 답할 땐 4.7K만 들고 들어간다.
13K가 빠진 셈이다.

## 교훈

> 프롬프트는 한 번에 다 넣지 마라.
> 매번 필요한 것만 들고 다니게 하라.

규칙을 늘리는 건 쉽다.
규칙을 *어디에 둘지* 결정하는 게 어렵다.

매뉴얼이 두꺼워질수록, 봇은 천천히 답한다.
사람도 그렇잖아.
